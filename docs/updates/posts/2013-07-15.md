# ocf.berkeley.edu now points to web server
After nearly three years of deprecating the <span style="font-family: "Courier New",Courier,monospace;">ocf.berkeley.edu</span> hostname for SSH/SFTP access to shell accounts,  we have decided to point <span style="font-family: "Courier New",Courier,monospace;">ocf.berkeley.edu</span> to the web server.<br />
<br />
The good news is that <span style="font-family: "Courier New",Courier,monospace;">ocf.berkeley.edu</span> in web browsers will seamlessly redirect (302 permanent redirect) to <span style="font-family: "Courier New",Courier,monospace;">www.ocf.berkeley.edu</span>. Less keystrokes, less to remember.<br />
<br />
The bad news is that you cannot access your shell account using the hostname <span style="font-family: "Courier New",Courier,monospace;">ocf.berkeley.edu</span>. You will need to use <span style="font-family: "Courier New",Courier,monospace;">ssh.ocf.berkeley.edu</span> as has been listed in <a href="http://wiki.ocf.berkeley.edu/services/shell/">our wiki documentation</a>.<br />
<br />
We believe that the compromise is worth it especially in light of the long transition period.
